Étape 1: Installation de Samba



Q1: Installez Samba en utilisant la commande suivante : 

sudo apt-get update 

sudo apt-get install samba samba-common-bin 

  

Correction Q1: La commande ci-dessus met à jour les sources de paquets de votre système Debian puis installe les paquets Samba et Samba Common Bin.



Étape 2: Configuration de Samba


Q2: Faites une copie de sauvegarde du fichier de configuration original de Samba : 

sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak 

Correction Q2: La commande ci-dessus crée une copie de sauvegarde du fichier de configuration de Samba. 



Q3: Éditez le fichier de configuration de Samba avec votre éditeur de texte préféré, par exemple :  

sudo nano /etc/samba/smb.conf

Correction Q3: La commande ci-dessus ouvre le fichier de configuration de Samba avec l'éditeur de texte Nano.


 

Q4: Configurez un partage public dans le fichier de configuration de Samba en ajoutant les lignes suivantes à la fin du fichier : 

[public] 

   path = /srv/samba/public 

   public = yes 

   writable = yes 

   guest ok = yes 

   guest only = yes 

   force create mode = 0777 

   force directory mode = 0777 

   browseable = yes  

Correction Q4: Les lignes ajoutésudes ci-dessus définissent un partage nommé "public" avec le chemin spécifié. Les options permettent à n'importe qui d'accéder, de lire et d'écrire dans ce partage sans authentification.




Q5: Créez le répertoire de partage et donnez-lui les permissions appropriées : 

sudo mkdir -p /srv/samba/public 

sudo chown nobody:nogroup /srv/samba/public 

sudo chmod 0777 /srv/samba/public  

Correction Q5: Ces commandes créent le répertoire de partage, le rendent accessible par l'utilisateur "nobody" et le groupe "nogroup", et donnent des permissions en lecture, écriture et exécution à tous les utilisateurs.




Étape 3: Redémarrage de Samba et tests 
 

Q6: Redémarrez le service Samba pour appliquer les modifications : 
                 Sudo systemctl restart smbd 

Correction Q6: La commande ci-dessus redémarre le service Samba pour prendre en compte les modifications apportées au fichier de configuration.



Q7: Vérifiez que le partage est accessible depuis un client Windows en entrant dans l'explorateur de fichiers l'adresse suivante : 

\\<adresse_IP_de_votre_serveur_debian>\public 

Correction Q7: En utilisant l'explorateur de fichiers Windows et en entrant l'adresse fournie, vous devriez accéder au partage public que vous avez configuré sur votre serveur Debian. Vous pouvez maintenant lire, écrire et modifier des fichiers dans ce partage sans authentification



Cours  : Accèdes au partage avec machine linux


PS : Connecter vous avec le serveur samba et autoriser l’usage des ports sur le firewall 

sudo ufw allow 445/tcp 

sudo ufw allow 139/tcp 



1: Connecter vous avec un autre machine et utiliser smbclient : 

 

sudo apt-get update 

sudo apt-get install smbclient 

 

Cette étape vous permet de vérifier si le problème est lié au processus de montage ou à la connexion elle-même. Installez smbclient si vous ne l'avez pas déjà fait.  



2: Essayez de vous connecter au partage Samba   

smbclient //192.168.232.102/public -N

 

Si vous pouvez vous connecter avec succès en utilisant smbclient, cela signifie que le problème est probablement lié au processus de montage. 




3:  Essayez de monter le partage Samba en utilisant l'adresse IP et en spécifiant explicitement la version du protocole SMB. Par exemple, pour utiliser la version 3.0 du protocole SMB, utilisez la commande suivante : 

 

sudo mount -t cifs //192.168.232.102/public ~/samba-public -o guest,uid=$(id -u),gid=$(id -g),vers=3.0
