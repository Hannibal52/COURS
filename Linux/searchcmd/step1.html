En cas de boulette (filaire non géré): 

Vérifiez que le gestionnaire de réseau NetworkManager est installé et activé. Pour cela, exécutez les commandes suivantes : 

sudo apt-get install network-manager 

sudo systemctl enable --now NetworkManager 

Ouvrez le fichier de configuration /etc/network/interfaces avec un éditeur de texte, par exemple : 

sudo nano /etc/network/interfaces 

Commentez la partie concernant votre interface ens33 en ajoutant un # devant chaque ligne, pour que NetworkManager prenne en charge cette interface :

#auto ens33 

#iface ens33 inet static 

#  address 192.168.232.110 

#  netmask 255.255.255.0 

#  gateway 192.168.232.2 

#  dns-nameservers 8.8.8.8 8.8.4.4


Enregistrez les modifications et fermez le fichier en appuyant sur Ctrl+X, puis Y et Entrée.

Redémarrez le service NetworkManager avec la commande suivante : 

sudo systemctl restart NetworkManager

Vérifiez si l'interface est maintenant gérée par NetworkManager. Vous pouvez utiliser l'outil nmtui pour configurer votre interface réseau. Pour cela, exécutez la commande suivante : 

sudo nmtui 

et suivez les etapes  

ping 8.8.8.8

File to Trash 

 

On crée un fichier filetotrash pour le test

Pour supprimer un fichier en le déplaçant dans la corbeille à l'aide de la commande gio trash

Pour le recupérer on utilise la même commande  

  gio restore filetotrash
