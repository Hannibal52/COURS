GLPI / CMS 

 

  • Mettre à jour le système : 
     

  • sudo apt update 

  • sudo apt upgrade 
     

  • Installer Apache : 
     

  • sudo apt install apache2 
     

  • Installer MariaDB (système de gestion de base de données) : 
     

  • sudo apt install mariadb-server 
     

  • Sécuriser l'installation de MariaDB : 
     

  • sudo mysql_secure_installation 
     

  • Répondez aux questions pour configurer un mot de passe root pour MariaDB et suivre les recommandations de sécurité. 
     

  • Créer la base de données et l'utilisateur pour GLPI : 
     

  • sudo mysql -u root -p  
    Entrez le mot de passe root de MariaDB. 

CREATE DATABASE glpi; 

CREATE USER 'glpiuser'@'localhost' IDENTIFIED BY 'password'; 

GRANT ALL PRIVILEGES ON glpi.* TO 'glpiuser'@'localhost'; 

FLUSH PRIVILEGES; 

Exit 
 
 

  • Remplacez 'password' par un mot de passe sécurisé pour l'utilisateur glpiuser. 

  • Installer PHP et les extensions nécessaires : 

 

  • sudo apt install php libapache2-mod-php php-mysql php-curl php-gd php-intl php-mbstring php-xml php-apcu php-ldap php-xmlrpc php-cas php-bz2 
     

  • Télécharger GLPI : 

 

  • cd /tmp 

  • wget https://github.com/glpi-project/glpi/releases/download/10.0.7/glpi-10.0.7.tgz 
     

  • Vérifiez la dernière version de GLPI sur https://github.com/glpi-project/glpi/releases et ajustez l'URL si nécessaire. 
     

  • Extraire et déplacer GLPI : 
     

  • tar xvf glpi-10.0.7.tgz 

  • sudo mv glpi /var/www/html/ 
     

  • Modifier les permissions : 

 

  • sudo chown -R www-data:www-data /var/www/html/glpi 
     

  • Configurer les virtual hosts pour Apache et GLPI : 

  • Créez un fichier de configuration pour chaque virtual host : 

 

  • sudo nano /etc/apache2/sites-available/glpi.conf 

  • sudo nano /etc/apache2/sites-available/apache_cms.conf 
     

  • Ajoutez la configuration suivante pour chaque fichier, en remplaçant ServerName et DocumentRoot par les valeurs appropriées : 

 

Pour le virtual host GLPI (glpi.conf) : 

<VirtualHost *:80> 

    ServerName glpi.example.com 

    DocumentRoot /var/www/html/glpi 

    <Directory /var/www/html/glpi> 

        Options FollowSymLinks 

        AllowOverride All 

        Require all granted 

    </Directory> 

    ErrorLog ${APACHE_LOG_DIR}/glpi_error.log 

    CustomLog ${APACHE_LOG_DIR}/glpi_access.log combined 

</VirtualHost> 

 

 

 

 

Pour le virtual host Apache CMS (apache_cms.conf) : 

<VirtualHost *:80> 

    ServerName cms.example.com 

    DocumentRoot /var/www/html/cms 

    <Directory /var/www/html/cms> 

        Options FollowSymLinks 

        AllowOverride All 

        Require all granted 

    </Directory> 

    ErrorLog ${APACHE_LOG_DIR}/cms_error.log 

  CustomLog ${APACHE_LOG_DIR}/cms_access.log combined 

        </VirtualHost> 

 

 

  • Activez les virtual hosts et redémarrez Apache : 

 

  • sudo a2ensite glpi.conf 

  • sudo a2ensite apache_cms.conf 

  • sudo systemctl restart apache2 
     

  • Configurer le pare-feu (si vous utilisez ufw) pour autoriser le trafic HTTP : 

 

  • sudo ufw allow 80/tcp 

  • sudo ufw allow 443/tcp 

 

  • Terminez l'installation de GLPI via le navigateur : 

 
Accédez à l'URL de votre serveur GLPI (par exemple, http://glpi.example.com) et suivez les instructions pour terminer l'installation. Utilisez les informations de la base de données et de l'utilisateur que vous avez créées précédemment.
