Tutoriel: Commandes de Windows Server 2019

Commande: netdom query fsmo

Définition
La commande 'netdom query fsmo' est utilisée pour afficher les maîtres d'opérations de rôle unique flexible (FSMO) dans un domaine Active Directory.
Usage
Ouvrez une invite de commandes en tant qu'administrateur, puis tapez 'netdom query fsmo' et appuyez sur Entrée. Les rôles FSMO actuels seront affichés.

Commande: ntdsutil

Définition
'ntdsutil' est un utilitaire de ligne de commande qui permet à un administrateur de gérer et de réparer Active Directory. Il peut être utilisé pour effectuer diverses tâches, y compris la gestion des rôles FSMO.
Usage
  • Active Instance NTDS: Utilisé pour définir l'instance Active Directory actuellement active.
  • Roles: Permet d'accéder à l'interface de gestion des rôles FSMO.
  • Connections: Permet de gérer les connexions à d'autres contrôleurs de domaine.
  • Connect to server localhost: Connecte l'utilitaire à un contrôleur de domaine spécifique (dans ce cas, le serveur local).
  • Quit FSMO: Quitte le menu de gestion des rôles FSMO.
  • Help: Affiche l'aide pour les commandes disponibles.
  • Transfer: Transfère un rôle FSMO à un autre contrôleur de domaine sans le supprimer du contrôleur actuel.
  • Seize Infrastructure Master: Si le transfert d'un rôle FSMO échoue, cette commande peut être utilisée pour "saisir" le rôle de force.
  • FSMO a échoué: le transfert a déjà été effectué: Ce message d'erreur signifie que le rôle FSMO que vous essayez de transférer a déjà été transféré.
  • Help Seize Naming Master: Affiche de l'aide pour la commande "Seize Naming Master", qui est utilisée pour saisir le rôle FSMO du maître de schéma si le transfert normal échoue.


  • Scenario avec une seule ligne de commande ntdsutil:
  • ntdsutil "activate instance ntds" "ifm" "create full C:\Backup"

    Cette commande active l'instance ntds, entre dans le menu IFM (pour les médias d'installation de la forêt), puis crée une sauvegarde complète dans le répertoire C:\Backup
