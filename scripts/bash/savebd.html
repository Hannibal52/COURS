Sauvegarder une base de données MySQL ou MariaDB

Voici un exemple de script Bash pour sauvegarder une base de données MySQL ou MariaDB sur un serveur Linux et compresser la sauvegarde dans un fichier gzippé. 



#!/bin/bash

# Paramètres de configuration
DB_NAME="nom_de_la_base_de_donnees"
DB_USER="utilisateur_de_la_base_de_donnees"
DB_PASSWORD="mot_de_passe_de_la_base_de_donnees"
BACKUP_DIR="/chemin/vers/le/repertoire/backup"
FILE_PREFIX="backup_db_"

# Récupérer la date actuelle
DATE=$(date +%Y%m%d_%H%M%S)

# Créer un nom de fichier pour la sauvegarde
BACKUP_FILE="${BACKUP_DIR}/${FILE_PREFIX}${DATE}.sql.gz"

# Sauvegarder la base de données et compresser le fichier de sauvegarde
echo "Création de la sauvegarde : ${BACKUP_FILE}"
mysqldump --user="${DB_USER}" --password="${DB_PASSWORD}" "${DB_NAME}" | gzip > "${BACKUP_FILE}"

# Vérifier si la création de la sauvegarde a réussi
if [ $? -eq 0 ]; then
  echo "Sauvegarde créée avec succès : ${BACKUP_FILE}"
else
  echo "Échec de la création de la sauvegarde. Veuillez vérifier les erreurs ci-dessus."
fi


Assurez-vous de remplacer les valeurs des variables DB_NAME, DB_USER, DB_PASSWORD et BACKUP_DIR par les informations appropriées pour votre base de données et le répertoire de sauvegarde. Le script crée ensuite une sauvegarde de la base de données à l'aide de mysqldump et compresse la sauvegarde dans un fichier gzippé avec un nom de fichier basé sur la date et l'heure actuelles.


Vous pouvez exécuter ce script manuellement ou le planifier avec cron pour effectuer des sauvegardes régulières de votre base de données.
