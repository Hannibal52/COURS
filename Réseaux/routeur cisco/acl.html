ACL Routeur cisco

Dans ce tutoriel, vous apprendrez à configurer les listes de contrôle d'accès (ACL) sur les routeurs Cisco à l'aide des commandes appropriées. Les ACL sont utilisées pour filtrer le trafic réseau et appliquer des règles de sécurité spécifiques.

Étape 1: Accédez au mode de configuration global

Pour commencer, connectez-vous au routeur Cisco et accédez au mode de configuration global en utilisant les commandes suivantes:

enable
configure terminal

Étape 2: Créez l'ACL

Ensuite, créez une ACL standard ou étendue en utilisant les commandes suivantes. Remplacez "ACL_NUM" par le numéro de l'ACL que vous souhaitez créer (1-99 pour les ACL standard et 100-199 pour les ACL étendues).

access-list ACL_NUM [remark DESCRIPTION]

Exemple :

access-list 101 remark "Blocage du trafic HTTP"

Étape 3: Ajoutez des règles à l'ACL

Ajoutez des règles à l'ACL pour permettre ou refuser le trafic en fonction de l'adresse IP source, de l'adresse IP de destination, du protocole et d'autres critères. Remplacez "ACTION" par "permit" ou "deny", et "CONDITION" par les critères de filtrage appropriés.

access-list ACL_NUM ACTION CONDITION

Exemple :

access-list 101 deny tcp any host 192.168.1.10 eq 80
access-list 101 permit ip any any

Étape 4: Appliquez l'ACL à une interface

Appliquez l'ACL à une interface spécifique pour filtrer le trafic entrant ou sortant. Remplacez "INTERFACE" par le nom de l'interface (par exemple, "FastEthernet0/0") et "DIRECTION" par "in" ou "out".

interface INTERFACE
ip access-group ACL_NUM DIRECTION
exit

Exemple :

interface FastEthernet0/0
ip access-group 101 in
exit

Étape 5: Vérifiez la configuration

Vérifiez la configuration de l'ACL en utilisant les commandes suivantes :

show access-lists
show ip interface INTERFACE

Exemple :

show access-lists
show ip interface FastEthernet0/0

Étape 6: Sauvegardez la configuration

Pour sauvegarder la configuration actuelle du routeur, utilisez la commande suivante :

write memory

Voilà ! Vous avez maintenant créé et appliqué une ACL sur un routeur Cisco en utilisant les commandes appropriées. N'hésitez pas à ajuster les règles de l'ACL en fonction de vos besoins spécifiques.
