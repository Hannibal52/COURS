Tutoriel de configuration de sécurité de port Cisco

Introduction

Dans ce tutoriel, nous allons apprendre comment configurer la sécurité de port sur un switch Cisco en utilisant la ligne de commande.

Configuration de port

  1. interface <interface-name>
  2. switchport mode access
  3. switchport port-security
  4. switchport port-security maximum <max-count>
  5. switchport port-security mac-address <mac-address>
  6. switchport port-security mac-address sticky
  7. switchport port-security violation <violation-mode>
  8. copy running-config startup-config

Assurez-vous de remplacer <interface-name>, <max-count>, <mac-address> et <violation-mode> par les valeurs appropriées pour votre configuration.

Violations de port

  • Shutdown : met le port en état de shutdown si une violation se produit. Cette option désactive le port et envoie une notification SNMP pour signaler la violation.
  • Protect : permet uniquement les adresses MAC autorisées sur le port et enregistre les tentatives de connexion non autorisées dans les journaux. Cette option envoie également des alertes SNMP pour signaler les tentatives de connexion non autorisées.
  • Restrict : permet uniquement les adresses MAC autorisées sur le port et envoie des alertes SNMP pour les tentatives de connexion non autorisées. Cette option ne déclenche pas de violation de port et ne désactive pas le port.

Exemple

interface FastEthernet0/1
switchport mode access
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address AAAA.BBBB.CCCC
switchport port-security mac-address sticky
switchport port-security violation shutdown

Cet exemple montre une configuration de port complète pour la sécurité de port, y compris une adresse MAC statique et une violation de port en mode shutdown.
