Configuration de VLAN, Trunk, Spanning Tree et EtherChannels

Étape 1 : Création des VLAN 10 et 20 sur chaque switch

Switch1, Switch2, Switch3 :

enable
configure terminal
vlan 10
name VLAN10
exit
vlan 20
name VLAN20
exit

Étape 2 : Attribution des ports aux VLAN

Switch1, Switch2 et Switch3 :

interface FastEthernet0/10
switchport mode access
switchport access vlan 10
exit
interface FastEthernet0/10
switchport mode access
switchport access vlan 20
exit

Étape 3 : Configuration des liens trunk

Switch1 :

interface range FastEthernet0/3 - 4
switchport mode trunk
switchport trunk allowed vlan 10,20
exit
interface range FastEthernet0/5 - 6
switchport mode trunk
switchport trunk allowed vlan 10,20
exit

Switch2 :

interface range FastEthernet0/3 - 4
switchport mode trunk
switchport trunk allowed vlan 10,20
exit
interface range FastEthernet0/1 - 2
switchport mode trunk
switchport trunk allowed vlan 10,20
exit

Switch3 :

interface range FastEthernet0/1 - 2
switchport mode trunk
switchport trunk allowed vlan 10,20
exit
interface range FastEthernet0/5 - 6
switchport mode trunk
switchport trunk allowed vlan 10,20
exit

Étape 4 : Configuration du Spanning Tree Protocol (STP)

Switch1 :

        spanning-tree vlan 10 root primary

Switch2 :

        spanning-tree vlan 20 root primary

Étape 5 : Configuration des EtherChannels

Switch1 :

    interface port-channel 2
    switchport mode trunk
    exit
    interface port-channel 3
    switchport mode trunk
    exit

Switch2 :

    interface port-channel 1
    switchport mode trunk
    exit
    interface port-channel 2
    switchport mode trunk
    exit

Switch3 :

    interface port-channel 1
    switchport mode trunk
    exit
    interface port-channel 3
    switchport mode trunk
    exit

Étape 6 : Vérification des configurations

Pour vérifier les différentes configurations, utilisez les commandes suivantes :

Vérification des VLAN :

show vlan brief

Vérification des liens trunk :

show interfaces trunk

Vérification du Spanning Tree Protocol :

show spanning-tree

Vérification des EtherChannels :

show etherchannel summary

Bonnus :

Après avoir terminé ces étapes, les switches devraient être configurés avec les VLAN, les liens trunk, le Spanning Tree et les EtherChannels. Vous pouvez alors connecter les PCs aux ports d'accès des VLAN correspondants.




Après avoir effectué toutes les configurations et connecté les PCs, il est important de tester la connectivité entre les PCs au sein des mêmes VLAN et de vérifier qu'il n'y a pas de connectivité entre les PCs de différents VLAN.



Utilisez la commande "ping" pour vérifier la connectivité entre les PCs. Assurez-vous que les PCs aient des adresses IP appropriées et qu'ils appartiennent au même sous-réseau pour chaque VLAN. Par exemple, pour le VLAN 10, vous pouvez attribuer des adresses IP telles que 10.0.10.2/24 et 10.0.10.3/24, et pour le VLAN 20, vous pouvez attribuer des adresses IP telles que 10.0.20.2/24 et 10.0.20.3/24.
