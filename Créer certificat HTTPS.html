Créer certificat HTTPS

June 16, 2023

Pour générer un certificat SSL gratuit à partir de LetsEncrypt pour votre site Web hébergé sur une VM Azure avec Debian 11 et Apache2, vous pouvez suivre les étapes ci-dessous:

Avant de commencer, assurez-vous que le nom de domaine de votre site Web est bien configuré et pointe vers l'adresse IP publique de votre VM Azure.

  1. Installer Certbot: Certbot est un client pour LetsEncrypt qui permet de demander des certificats. Pour l'installer, connectez-vous à votre VM via SSH et exécutez les commandes suivantes:

    bash
    sudo apt-get update
sudo apt-get install certbot python3-certbot-apache

  2. Demander un certificat: Vous pouvez demander un certificat SSL en exécutant la commande suivante. Remplacez yourdomain.com avec votre nom de domaine:

    bash
    sudo certbot --apache -d yourdomain.com

    Si vous avez plusieurs noms de domaine ou sous-domaines, vous pouvez les ajouter en incluant d'autres arguments -d, par exemple -d www.yourdomain.com -d sub.yourdomain.com.

  3. Suivre les instructions de Certbot: Certbot vous demandera de fournir une adresse e-mail pour les notifications de renouvellement et les alertes de sécurité, et vous devrez également accepter les termes du service. Ensuite, Certbot vérifiera que vous contrôlez le nom de domaine en plaçant un fichier sur votre serveur web et en essayant de le télécharger.

  4. Choisir la configuration HTTPS: Certbot vous demandera si vous souhaitez rediriger tout le trafic HTTP vers HTTPS. Choisissez l'option qui vous convient le mieux.

Après ces étapes, Certbot aura généré un certificat SSL pour votre site Web et aura configuré Apache pour utiliser ce certificat. Les certificats LetsEncrypt sont valides pendant 90 jours, mais Certbot tentera de les renouveler automatiquement 30 jours avant leur expiration.

Pour exporter ce certificat et l'utiliser dans votre Application Gateway dans Azure, vous pouvez le trouver dans le dossier /etc/letsencrypt/live/yourdomain.com/. Les fichiers fullchain.pem et privkey.pem sont respectivement votre certificat public et votre clé privée. Vous devrez les convertir en un seul fichier .pfx pour pouvoir l'uploader dans Azure. Voici comment vous pouvez le faire:

bash
sudo openssl pkcs12 -export -out certificate.pfx -inkey /etc/letsencrypt/live/yourdomain.com/privkey.pem -in /etc/letsencrypt/live/yourdomain.com/fullchain.pem

Après avoir créé votre fichier .pfx, vous pouvez l'uploader dans votre Application Gateway dans Azure.
