Jenkins et Ansible sont deux outils qui peuvent être utilisés ensemble pour automatiser différents aspects du déploiement, de la gestion et de l'intégration continue (CI) et de la livraison continue (CD) des applications. Voici pourquoi vous pourriez envisager d'installer Jenkins pour travailler avec Ansible en quelques mots: Automatisation CI/CD: Jenkins est l'un des outils d'intégration continue les plus populaires. Il peut déclencher des playbooks Ansible à chaque push de code, garantissant que chaque modification est testée et déployée automatiquement. Orchestration: Alors qu'Ansible est idéal pour configurer et déployer, Jenkins peut orchestrer une séquence complexe d'événements, tels que la construction d'un logiciel, le test, puis le déploiement via Ansible. Interface utilisateur: Jenkins fournit une interface utilisateur graphique pour la planification de tâches, la visualisation des logs, et le déclenchement manuel de builds, ce qui peut rendre plus facile la gestion des tâches automatisées avec Ansible pour certains utilisateurs. Plugins: Il existe des plugins Jenkins spécifiques pour Ansible qui facilitent l'intégration et l'utilisation des deux outils ensemble. Traçabilité: Avec Jenkins, vous pouvez avoir un historique complet des exécutions, y compris quand et où Ansible a été exécuté, et avec quels résultats. En résumé, l'utilisation de Jenkins avec Ansible permet d'obtenir une solution complète d'intégration et de déploiement continu, où Jenkins s'occupe de l'orchestration et Ansible de l'exécution.


Installation Jenkins

ceci est un script pour debian 11

#!/bin/bash # Mise à jour du systèmep sudo apt update && sudo apt upgrade -y # Installation des prérequis sudo apt install -y wget gnupg2 # Ajout de la clé GPG de Jenkins wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add - # Mise à jour des paquets sudo apt update # Recherche des paquets OpenJDK (ceci affichera une liste, mais n'installera rien) apt-cache search openjdk # Installation d'OpenJDK 11 sudo apt-get install -y openjdk-11-jre openjdk-11-jdk # Installation d'OpenJDK 8 (si vous voulez vraiment les deux versions) sudo apt-get install -y openjdk-8-jre openjdk-8-jdk # Vérification de la version de Java java -version # Ajout du dépôt de Jenkins sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list' # Mise à jour des paquets sudo apt update # Installation de Jenkins sudo apt install -y jenkins # Démarrage du service Jenkins sudo systemctl start jenkins # Affichage du statut de Jenkins sudo systemctl status jenkins # Affichage du mot de passe initial pour Jenkins echo "Voici votre mot de passe initial pour Jenkins:" sudo cat /var/lib/jenkins/secrets/initialAdminPassword # Obtention de l'adresse IP privée private_ip=$(hostname -I | awk '{print $1}') echo "Visitez http://$private_ip:8080/ pour configurer Jenkins."
chmod +x install_jenkins.sh ./install_jenkins.sh
