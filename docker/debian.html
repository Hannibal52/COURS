Tutoriel : Docker et Kubernetes Sur Linux (debian)

Étape 1: Installer Docker

  1. Mettez à jour votre liste de packages existante : 
    sudo apt-get update
  2. Installez les packages nécessaires pour que apt utilise des dépôts sur HTTPS : 
    sudo apt-get install apt-transport-https ca-certificates curl gnupg lsb-release
  3. Ajoutez la clé GPG officielle de Docker : 
    curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
  4. Configurez le dépôt stable de Docker : 
    echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  5. Mettez à jour votre liste de packages à nouveau : 
    sudo apt-get update
  6. Installez Docker Engine : 
    sudo apt-get install docker-ce docker-ce-cli containerd.io
  7. Vérifiez que Docker est correctement installé en exécutant : 
    sudo docker run hello-world

Étape 2: Installer Kubernetes

Minikube : Minikube est un outil qui permet de lancer Kubernetes localement. Minikube lance un cluster Kubernetes à nœud unique dans une machine virtuelle sur votre ordinateur.
  1. Installez Minikube : 
    curl -Lo minikube https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
    chmod +x minikube
    sudo mv minikube /usr/local/bin/
Kubectl : Kubectl est une ligne de commande pour contrôler les clusters Kubernetes. C'est l'outil principal de communication entre l'utilisateur et le cluster.
  1. Installez Kubectl : 
    curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
    chmod +x ./kubectl
    sudo mv ./kubectl /usr/local/bin/kubectl
  2. Démarrez Minikube : 
    minikube start --driver=docker
  3. Vérifiez que Minikube est correctement installé en exécutant : 
    minikube status
