Tutoriel Ansible Vault

Ansible Vault est un outil intégré à Ansible permettant de chiffrer des données sensibles afin de les stocker de manière sécurisée.

1. Création d'un secret avec Ansible Vault

ansible-vault create secret.yml

Cette commande vous demandera de saisir un mot de passe. Ce mot de passe sera utilisé pour chiffrer et déchiffrer le fichier.


2. Éditer un fichier chiffré

ansible-vault edit secret.yml

Lors de l'édition, vous serez invité à saisir le mot de passe que vous avez défini lors de la création.


3. Chiffrer un fichier existant

ansible-vault encrypt existingfile.yml

4. Déchiffrer un fichier

ansible-vault decrypt secret.yml

5. Voir le contenu d'un fichier chiffré sans le déchiffrer

ansible-vault view secret.yml

6. Utiliser un fichier chiffré dans un playbook

Pour exécuter un playbook qui utilise des variables chiffrées avec Ansible Vault, utilisez la commande suivante:

ansible-playbook --ask-vault-pass playbook.yml

Il vous sera demandé d'entrer le mot de passe Vault pour déchiffrer les données pendant l'exécution.


7. Utiliser un mot de passe depuis un fichier

Plutôt que d'utiliser --ask-vault-pass à chaque fois, vous pouvez stocker le mot de passe Vault dans un fichier (assurez-vous que ce fichier est sécurisé!) et référencer ce fichier lors de l'exécution du playbook.

ansible-playbook --vault-password-file /chemin/vers/fichier/password.txt playbook.yml

8. Conseils

  • Toujours stocker les fichiers chiffrés avec Vault dans un système de contrôle de version (comme Git).
  • Ne jamais stocker le mot de passe Vault dans un système de contrôle de version ou un emplacement accessible publiquement.
